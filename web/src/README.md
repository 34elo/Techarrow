# FSD base structure

- `app` - app initialization, providers, global styles, routing entrypoint.
- `processes` - long-running user flows across pages.
- `pages` - route-level screens and composed page UI.
- `widgets` - large independent UI blocks for pages.
- `features` - user scenarios and actions.
- `entities` - domain entities and business models.
- `shared` - reusable libs, UI-kit, API clients, stores, config.
